## 1) ReLU in plain English: the gate that decides if your signal matters

ReLU stands for **Rectified Linear Unit** and it’s basically:

**ReLU(x) = max(0, x)**

Meaning:
- if the input signal is weak/negative → output becomes **0** (ignored)
- if the input is strong/positive → output passes through and grows **linearly**

In ad terms, ReLU behaves like a **bouncer**:
- show up with low energy (weak **CTR**, low conversion density, noisy feedback) → you don’t get in
- show up with clear demand (strong engagement + conversions) → you get prioritized

**Why this matters:** ReLU creates **non‑linear outcomes**.  
Below threshold = you feel invisible.  
Above threshold = performance can improve *fast* because delivery, placements, and feedback loops become stronger.

## 2) Why Meta’s MLP makes ads “winner‑take‑more” (not winner‑take‑all)

Meta’s ranking/prediction stack learns **hierarchical features** (simplified):
- **Layer 1:** basic signals (impressions, clicks, video watch, add‑to‑cart)
- **Layer 2:** interaction patterns (time‑to‑click, repeat engagement, device × placement behaviors)
- **Layer 3:** high‑intent behavior (purchase probability, value proxies, retention likelihood)

The key: once your campaign produces strong signals, the model predicts you’re “safe” (good user experience + good outcomes). That tends to produce:
- better auction outcomes
- lower **CPMs** (because your predicted value rises)
- more stable delivery (because feedback is clearer)

So your job is not to “hack” Meta.  
Your job is to **feed the model clean, high‑value patterns early**.

## 3) What “below threshold” looks like in Ads Manager

If you’re below the activation threshold, you’ll often see:
- Learning drag or **Learning Limited**
- unstable **CPA** / **ROAS** swings day‑to‑day
- low delivery (spend can’t scale even when budget is available)
- high CPMs despite decent creative
- scattered conversions across many ad sets (no concentration of signal)

## 4) The core strategy: Go deep before you go wide

Most advertisers try to reduce risk by spreading budget across many ad sets.

In a ReLU‑like world, that often backfires.

**Because thin budget = thin signal = below threshold.**

### The “ReLU‑friendly” budget rule
- Fewer ad sets (or fewer ad groups inside one campaign)
- More budget per optimization unit
- Clear objective (Conversions / Purchases, not mixed intent)
- Clean conversion path (Ad → Landing Page → Offer → Checkout)
- Clean tracking (Pixel + **Conversions API (CAPI)**) so Meta sees real outcomes

### A note on CBO vs ABO (and why it matters here)
Whether you use:
- **ABO** (ad set budgets), or  
- **CBO** (campaign budget optimization),

…the rule stays the same: **make sure the thing you’re optimizing (ad set / ad group / campaign) gets enough budget to generate dense feedback.**

## 5) A 3‑phase launch algorithm to “cross the threshold” early

### Phase 1 — Warm‑up (Days 1–7): build signal density
Goal: create obvious winners.
- start with your **best creative** (highest CTR + strongest conversion rate)
- use your **highest‑intent audience** first (customers / high‑value retargeting / top lookalikes)
- if you’re e‑com, start with the cleanest SKU/offer (don’t confuse the model with 10 offers at once)
- consider slightly more aggressive bids or cost controls *only if* you can maintain quality
- keep variables limited: don’t change 10 things at once

### Phase 2 — Expansion (Days 8–14): widen carefully without collapsing quality
Goal: expand reach while staying above threshold.
- broaden audiences gradually (stack lookalikes, broaden interest, test broad)
- keep budget on winners; do NOT “rescue” losers too early
- maintain message consistency so the model learns your “signature”

### Phase 3 — Efficiency (Days 15+): harvest algorithmic preference
Goal: reduce CPA while keeping delivery stable.
- slowly lower bids/cost caps (if you use them)
- keep winning creatives in rotation; refresh without breaking the message
- scale budgets in steps, not spikes

## 6) How to use a Knowledge Graph + “bias agents” to do this autonomously

If you’re building automation (or using an autonomous platform like **MIZ OKI 3.0™**), the goal is simple:

**Encode the causal playbook, then bias execution toward fast activation.**

### Knowledge Graph (KG): what it should store
Store relationships like:
- **budget concentration → higher conversion density → stronger model confidence**
- **creative relevance → higher CTR → better auction outcomes**
- **signal consistency → faster learning → more stable delivery**
- **higher event match quality → better attribution → better optimization**

### Bias agents: what they do
- **Threshold Discovery Agent:** probes spend/bids to locate the “performance cliff”
- **Signal Amplification Agent:** concentrates budget into top segments and top creatives
- **Momentum Agent:** reinvests into winners while controlling volatility

Here’s a simplified concept (not platform‑specific):

```python
def relu_threshold_playbook(state):
    # state contains: age_days, cpa, roas, events_per_week, volatility, etc.
    if state.age_days <= 7:
        return {"action": "CONCENTRATE", "audience": "TOP_INTENT", "budget": "HIGH", "changes": "MINIMAL"}
    elif state.age_days <= 14 and state.is_above_threshold():
        return {"action": "EXPAND_SLOWLY", "audience": "BROADEN", "budget": "STEP_UP", "changes": "CONTROLLED"}
    else:
        return {"action": "HARVEST", "budget": "OPTIMIZE_EFFICIENCY", "changes": "SMALL_ITERATIONS"}
```

The point: **automation should enforce “go deep first,”** because the math of ReLU‑like systems punishes dilution.

## 7) Guardrails: what NOT to do (and what Meta rewards)

Don’t try to “trick” the system with:
- low‑quality traffic sources
- click farms or artificial engagement
- chaotic bid changes that destabilize learning
- mixed messages across too many angles at once

Meta rewards:
- consistent creative + offer alignment
- high‑quality landing pages
- clean conversion tracking (Pixel + Conversions API / CAPI)
- predictable patterns (so the model can confidently scale)
